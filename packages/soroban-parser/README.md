# `@sorotrace/soroban-parser`

Soroban XDR decoding and storage tier parsing for the SoroTrace platform.

## What This Package Does

When a Soroban smart contract runs on the Stellar blockchain, the network records everything that happened in a binary format called XDR (External Data Representation). This package converts that binary data into typed TypeScript objects that the rest of SoroTrace can work with.

Specifically, it decodes:

- **Function names and arguments** - what function was called, and with what inputs.
- **Return values** - what the contract returned, or whether it failed.
- **Storage entry changes** - which contract storage slots were created, updated, or deleted as a side-effect of the transaction.
- **Contract events** - structured log entries emitted by the contract during execution.

## Current Status

**Spike phase.** The production parser has not been built yet.

A technical spike has been completed to validate that `@stellar/stellar-sdk` v13 can decode all common Soroban XDR types without requiring a custom binary parser. The spike findings are documented in:

- [`spike/decode-spike.ts`](spike/decode-spike.ts) - the runnable spike script
- [`spike/spike-notes.md`](spike/spike-notes.md) - full findings, gaps, and recommended production approach

Production implementations will be added in follow-up issues once the spike findings are reviewed.

## Running the Spike

```bash
# From the repo root
pnpm --filter soroban-parser spike

# Local-only mode (no network required)
SPIKE_MODE=local pnpm --filter soroban-parser spike

# Override the hardcoded transaction hash
SPIKE_HASH=<txhash> pnpm --filter soroban-parser spike
```

The spike connects to `soroban-testnet.stellar.org`. You can find valid testnet transaction hashes at [stellar.expert/explorer/testnet](https://stellar.expert/explorer/testnet) - look for transactions with type `InvokeHostFunction`.

## Building

```bash
# From the repo root
pnpm --filter soroban-parser build

# Or from this directory
pnpm build
```

Output is written to `dist/`.

## Key Concepts

If you are new to Stellar and Soroban, these terms appear throughout the codebase:

**XDR** - the binary serialisation format Stellar uses to encode all data on-chain. Every transaction, storage entry, and event is XDR under the hood. The Stellar SDK handles encoding and decoding.

**ScVal** - "Smart Contract Value". The union type that represents any value in a Soroban contract: numbers, strings, booleans, addresses, maps, vectors, etc. `scValToNative()` in the SDK converts these to JavaScript native types.

**TransactionMeta** - the blob of data the network attaches to a transaction after it executes, recording all the side-effects: which storage entries changed and what contract events were emitted.

**ContractDataEntry** - a single storage slot for a Soroban contract. Has a `key` (ScVal), a `val` (ScVal), and a `durability` (`persistent` or `temporary`).

**InvokeHostFunction** - the Stellar operation type that runs a Soroban contract function. Not all Stellar transactions invoke contracts; this package is only concerned with those that do.

## Dependencies

- [`@stellar/stellar-sdk`](https://github.com/stellar/js-stellar-sdk) - official Stellar JavaScript SDK, used for all XDR types and the `scValToNative` decoder.
